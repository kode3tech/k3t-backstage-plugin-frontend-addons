# Internationalization (i18n) - Presentation

This document describes the internationalization improvements made to the Entity Addons Plugin presentation.

## Overview

The presentation now supports **Brazilian Portuguese (pt-BR)** and **English (en-US)** languages with full internationalization support.

## Features

### 1. Language Switcher
- **Location**: Fixed in top-right corner
- **Visual Design**: Matches the neon theme of the presentation
- **Icons**: Uses Font Awesome icons
- **Active State**: Highlighted with gradient and neon glow

#### Language Button Styling:
- Default: Semi-transparent with subtle border
- Hover: Border and text highlight with neon glow
- Active: Gradient background with glowing effect

### 2. Language Persistence
- Selected language is automatically saved to browser's `localStorage`
- Key: `presentation-language`
- Returns saved preference on page reload
- Falls back to browser language or Portuguese if not set

### 3. Automatic HTML Language Attribute
- The `<html>` tag `lang` attribute updates dynamically
- Supports screen readers and browser language detection
- Values: `pt-BR` or `en-US`

### 4. Complete Translation Coverage
All 17 slides are fully translated including:
- Slide titles and subtitles
- Feature descriptions
- Technical terms and annotations
- Navigation hints
- Code examples descriptions (as labels, not code itself)
- Questions and answers
- Call-to-action messages

## Translation Structure

### Translation Object
```javascript
const translations = {
    'pt-BR': { /* Portuguese translations */ },
    'en-US': { /* English translations */ }
}
```

### Key Categories
1. **Page Metadata**: Page title, navigation hints
2. **Slide Content**: All visible text in slides
3. **Technical Terms**: Feature names, annotations, components
4. **Instructions**: Step-by-step guides
5. **Metadata**: Questions, answers, benefits

## Implementation Details

### HTML Markup
- Elements to translate use `data-i18n` attribute: `<span data-i18n="key">Default Text</span>`
- Elements with HTML content use `data-i18n-html` attribute: `<p data-i18n-html="key">...</p>`

### JavaScript Functions
- `t(key)`: Translate function - returns translated string or key if not found
- `updatePageLanguage()`: Updates all page content and language switcher state
- `getSavedLanguage()`: Retrieves saved preference or detects browser language

### Language Detection Order
1. Check `localStorage` for saved preference
2. Check browser language (navigator.language)
3. Default to Portuguese (pt-BR)

## Usage

### Switching Languages
Simply click the language button in the top-right corner:
- **PT-BR** button: Switches to Brazilian Portuguese
- **EN-US** button: Switches to English (US)

### Customizing Translations

To add or modify translations:

1. Edit the `translations` object in the JavaScript section
2. Add new keys following the existing pattern
3. Add `data-i18n` or `data-i18n-html` attributes to elements
4. Call `updatePageLanguage()` after DOM updates

Example:
```javascript
translations['pt-BR']['myNewKey'] = 'Texto em PortuguÃªs';
translations['en-US']['myNewKey'] = 'English Text';

// In HTML:
// <span data-i18n="myNewKey">Default Text</span>
```

### Adding New Elements

1. Add the translation key to both language objects
2. Add `data-i18n="keyName"` to the HTML element
3. The translation will be applied when language is changed

## Performance Considerations

- Translations are loaded at page initialization
- Language switching is instantaneous (no page reload needed)
- All translations are inline (no external file requests)
- localStorage provides fast language preference retrieval

## Browser Compatibility

- Modern browsers (Chrome, Firefox, Safari, Edge)
- Requires ES6 support (const, arrow functions, template literals)
- localStorage support required for preference persistence
- Graceful fallback to default language if localStorage unavailable

## Troubleshooting

### Language Not Changing
- Check browser console for errors
- Verify `data-i18n` attributes are correctly set
- Clear localStorage and reload page

### Translations Not Showing
- Ensure translation key exists in both language objects
- Verify HTML element has `data-i18n` attribute
- Check browser console for missing translation warnings

### Preference Not Saving
- Enable localStorage in browser settings
- Check if browser is in private/incognito mode
- Verify localStorage quota not exceeded
